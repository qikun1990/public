    Day 28: Doing a Bot Test

    UPDATE: Before reading this lesson, please see this post:

    https://stmforum.com/forum/showthrea...l=1#post403158



    ***************

    ACTION

    I was planning to write a lesson on how to do this - but then Voluum made this post on how to conduct bot testing on Voluum.

    So I'll just direct you to that instead. If you have any questions, please feel free to ask in that thread! Karolina from Voluum is extremely helpful and active here on STM - I'm sure she can help us with questions!

    https://stmforum.com/forum/showthrea...oluum-Honeypot

    Here are more detailed instructions:

    https://doc.Voluum.com/en/honeypot.h...%5D%20Honeypot

    This feature is not available for the Entry plan though - it's only available for Basic and Advanced plans only. If you're on the Entry plan but would like to test this feature for 2 weeks, please see Karolina's post here:

    https://stmforum.com/forum/showthrea...l=1#post370013

    If you're on their Entry plan and would like to perform a bot test without using Voluum's Anti-Fraud feature - caurmen's bot-testing instructions will be useful:

    https://stmforum.com/forum/showthrea...live-campaigns

    I've explained how this method works in this post:

    https://stmforum.com/forum/showthrea...l=1#post338489

    ...and talked about how to execute this method here:

    https://stmforum.com/forum/showthrea...l=1#post337880

    ...and explained how to interpret the data here:

    https://stmforum.com/forum/showthrea...l=1#post337771
    @platinum has made a diagram to clarify further on how the method works:

    https://stmforum.com/forum/showthrea...l=1#post338511



    Warning: It would be messy (if even possible) to perform bot-testing while split-testing multiple offers and landers. I would recommend to either:

    1)Perform bot-testing FIRST using just a single offer and lander. Basically do this at the start of a campaign. Set up the campaign, wait for traffic to run, then keep refreshing stats and pausing placements that have received more than 50 impressions (as suggested by caurmen as being the threshold for statistically-significant data). Of course you can't keep doing this until ALL placements have reached 50 impressions - that would take days/weeks/months. Doing this for 15-30 minutes should cover the biggest placements.

    OR:

    2)Perform bot-testing AFTER you've cut down to a good lander+offer. Instead of just adding the bot-test code to the original lander, it would be less-confusing to duplicate and rename it and add the bot code to it, and add it to the tracker as another lander (with "bot" somewhere in the lander name to identify it as such). That way, next time you need to test offers using the same lander, you won't need to edit the lander to delete the bot code.

    Please note that this method is from 2017. Since then, the bots have evolved, especially with the launch of Chrome Driver which allows to pilot from a script a real chrome browser. This means that considering human/non-human as supporting JS/not-supporting JS is not necesarily that true today.

    That said, it depends of the volume you are generating. The profit plan also allows to use the API and the redirectless method, which are the reasons why I upgraded yesterday.

    If you do not want to upgrade, you can still use the same method, and instead of loading a tracker link, you point to a php script that register the the parameters campaign / placementid into a file, so that you can analyze them later. It won't be that convenient, but il allows you to get the data.

    In your landing:
    Code:
        u = "https://www.yourdomain.com/script.php?cpid=" + cpid + "&placementid=" + placementid;
    document.write('');
    (of course you have to define cpid and placementid before, for example by reading them from the URL of your page)

    PHP:
    Code:
        file_put_contents( 'bot.log', $_GET["cpid"] + " - " + $_GET["placementid"], FILE_APPEND );
