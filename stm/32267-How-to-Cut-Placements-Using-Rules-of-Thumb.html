    How to Cut Placements Using Rules of Thumb

    Some of you may have used this placement analysis tool here:

    https://stmforum.com/forum/showthread...Banners-Part-2

    From the accuracy standpoint, it's a fantastic tool (if I do say so myself ).

    However, in exchange for the high level of accuracy, efficiency is sacrificed. It takes quite a bit of time to collect enough data to arrive at a "verdict".

    Especially when running pop traffic - with the typical shorter life-span of pop camps, being able to cut placements faster can be more beneficial than being able to cut with more accuracy.

    So I made another tool to cut placements using rules of thumb.






    What It Can Do

    You can set one or more of the following cutting rules, and the tool will tell you which placements to cut:


    Rule 1: Cut Placements with CTR < Certain %
    Rule 2: Cut Placements with CTR > Certain %

    Placements with overly high or low CTR are often associated with a high percentage of bot traffic. You can specify "cut-off" CTRs , and the spreadsheet will list all the placements below the minimum CTR and above the maximum CTR you specify.

    You will need to specify a minimum number of Visits/Impressions. Of course the higher this number, the more accurate your CTR will be. Caurmen suggests that 30-50 impressions should be enough when running the bot test. Although we're not running a bot test here, 30-50 impressions should be a sufficient sample size to reflect an accurate CTR.

    Moreover, the spreadsheet will actually recommend to you the cut-off CTRs! The tool will tell you that if you had cut all placements with less than X% CTR or greater than Y% CTR, you would have made the maximum amount of profits based on current data.


    Rule 3: Cut Placements in Loss by X Payouts

    This is my personal favorite! I just did this a couple of days ago for one of my camps by cutting all placements in loss by 2x the payout, and it's been doing twice the CR (and would have done a lot more profits than before the cut, if I hadn't accidentally let my balance run out on that traffic source!)

    You can set the "X" to anything you like, depending on the particular situation. I've been seeing good results by setting it to 2 or 3. If you're getting a ton of traffic volume and want to get green faster, you could even set X to 1 or even 0.50 to cut very aggressively so that you can test offers/landers at lower cost. (When your camp gets green though, it's recommended to retest the blacklisted placements.)


    Rule 4: Cut Placements That Don't Convert

    If the cost exceeds X payouts without converting, cut the placement. Again, you can set this to any value. I like to set this to 2.




    How to Use It

    I've written instructions directly into the spreadsheet. But will include them below as well, along with extra comments.


    Step 1: Set Basic Values

    "Payout ($):" Enter your offer's payout amount if you're wanting to run Rules 3 and/or 4.

    "CPM ($):" Enter your bid in CPM. This is only necessary if you don't have the actual cost data per placement (which is usually downloaded from the traffic source, unless the TS can pass costs accurately back to the tracker).


    Step 2: Paste Placement Data

    "Placement Name", "Visits/Impressions", "Revenue", and "CTR" - these you'll probably get from your tracker.

    As for "Cost From Stats", you'll probably need to get these from your traffic source stats.

    If that's the case, you'll need a tool to match up your revenue and cost data - this is what the "Combine Rev. & Costs" sheet is for. Instructions on how to use that are on the sheet directly - pretty straight-forward to use.


    Step 3: Select Rules to be Applied

    Choose the rules you want to be applied by marking the highlighted cell beside each rule with an "x".


    Step 4: Set Values for Rules

    Pretty self-explanatory - enter values for each rule you've chosen to be applied.

    If you're using the FULL version of the tool, you'll see suggestions for the Minimum and Maximum Cut-off CTRs. If you choose to implement these suggested values, type them into the appropriate highlighted cells.

    Note regarding recommended Min. and Max. CTRs: The more data you have, the more accurate the recommended percentages will be. IMPORTANT: To use this feature, you should use stats that resulted from a SINGLE lander - otherwise your CTRs will be all over the place and the CTR recommended by the tool will be meaningless.

    So - when grabbing stats from your tracker, either 1)set the date range to when you've been running your best lander only, or 2)drill down into placement stats for the lander you've gathered the most traffic for.


    Step 5: Grab the List of Placement to Cut

    Last step would be to actually pause the placements listed by the tool, at the traffic source.

    Important: I can't figure out how to automatically sort the column containing the placements to cut, so that all the blank lines are removed. You can copy the entire column and paste it into an online empty line remover like this one:

    http://www.miniwebtool.com/remove-empty-lines/

    Or paste the placements onto another excel sheet and use the sort function.


    Although that sounded like a lot of steps, the tool is actually a lot easier to use than it looks.




    Important Notes

    -This tool is great for efficiency but isn't too accurate. If you're getting all or even a big portion of your traffic from several big placements, this would be the better tool to use:

    https://stmforum.com/forum/showthread...Banners-Part-2

    -Statistical significance still has a very important place. Please do not apply the same approach by cutting corners when testing offers and landers. There's a huge difference between cutting corners with thousands of small placements, and doing the same with something that can make or break your camp like the offer and lander.

    In fact, after a recent discussion with matuloo, I've decided to increase my "probability of being best" cutting threshold from 90% to 95% or sometimes even 100% when cutting offers and landers. You're encouraged to test the same.

    -Ideally, you should only include placement stats collected after you've arrived at your best offer+creatives+bid. However, every situation is different, so use this tool however you wish.

    -It is recommended that you save a list of the placements you're cutting somewhere, so that if you see a drop in performance after cutting them, you can re-introduce them.

    -Disclaimer: I cannot guarantee that this tool is free of bugs, or that it will help to increase your profits. I cannot be held responsible for any loss you may sustain, monetary or otherwise, as a result of using this tool.

    No - what matuloo and I were discussing, was the percentage of "probability of being best" when comparing landers using the peakconversion split-test calculator.

    You can download the full version of the spreadsheet above, paste some data you've already collected for a camp, and it will suggest CTR % you can cut off at.

    This suggestion is based purely on the data you feed into it. It's kind of like a backtester, where the tool will tell you "if you had cut all placements with a CTR of higher than X or lower than Y, then your profits would have been maximized".

    IMPORTANT NOTE: The CTR we're talking about is LANDING PAGE CTR, and NOT the percentage of BOT TRAFFIC! However, a very high or very low CTR can be indication of heavy bot traffic.

    Lander CTR can be dependent on a lot of variables, and the values can vary widely. Therefore you can't use a rule of thumb that says "anything above X or below Y is probably bad" and apply that to every lander. Even when using my tool above to suggest values for high and low CTR cutoff values, it would be appropriate to use stats for a SINGLE lander.

    To test for bot traffic please use caurmen's script:

    https://stmforum.com/forum/showthread...Your-Campaigns

    Basically it's up to you how you want to apply this tool.

    If you want to get more accurate results, you can collect more data over a longer period first, and then run the tool.

    Do keep in mind that the whole point of using rules of thumb, is so that you can cut placements faster, but with less accuracy. If you have a lot of data collected over a longer period, then you may as well use the statistics spreadsheet here:

    https://stmforum.com/forum/showthread...Banners-Part-2

    I would personally just ignore how many days I need to collect data for. If I set the rule of thumb to "cut placement when cost is great than two times the payout", I would cut it even if I only have a few hours' worth of data. Of course you can collect data for a few days before deciding, but that would be extra costs.

    Something to be careful about though, is that there may be delayed conversions. The visitor may leave the offer open and convert later on, and the affiliate network may not post back the conversion until later. Therefore, when you're dealing with high-traffic placements, it may be better to either collect more data first or put the camp on pause for a day or so to wait for the conversions to catch up, before looking at data and making cutting decisions. Again, if you want to be more accurate when dealing with high-traffic placements, you can use the stats spreadsheet I linked to above.


    e.g. placement A is in profit for day 1 and then it loose 10 x payout while day2. The next day (3) will be in profit as day1.
    Yes, there will be volatility in performance on different days. The question you need to ask yourself is "how volatile is the performance typically?"

    It's HIGHLY unlikely for a placement to be in loss by 10x payout one day, and be profitable the next day.

    Let's put it this way:

    -The higher the number of payouts you set the rule to, the more time and money you'll need to spend on cutting placements, but your chances of cutting potentially-profitable placements will be smaller.

    -The lower the number of payouts you set the rule to, the less time and money you'll need to spend on cutting placements, but your chances of cutting potentially-profitable placements will be higher.

    So it's really up to you. Based on experience, I find that 2x in loss works for me. I'm sure I must have cut some placements that could have been profitable if I had spent more money than that, but I don't care - because I've saved a lot of time and money by being less accurate! As always, we need to mind the 80/20 rule.

    As for raising the bid - that's a separate topic, and is a big one. I've had quite a big discussion with mrpayne on this - please see this post and the ones above it:

    https://stmforum.com/forum/showthread...l=1#post290662

    I may write a post on different placement testing approaches and how to tweak bids in a separate thread. Basically, here are several approaches I could think of:

    1)If you're dealing with a source like zeropark where you can change the bid for individual placements: my link to that post in mrpayne's thread applies.

    If you're dealing with sources that only allows you to set one bid per camp, you have a few options...

    2a)You can test several bids - low/average/high - and you can test more than 3 bids - basically see which one will give you the most profits, and leave it running at that bid. May need to retest from time to time (if your camp lives that long!) because the bidding landscape is always changing with competition etc.

    2b)You can run a camp at average bid, then start a higher-bid camp and a lower-bid camp, and move the good performers to the higher-bid camp and the not-profitable-enough-but-still-converting placements to the lower-bid camp.

    2c)You can start a camp at low bid, run some traffic to it and identify which placements are getting good traffic levels. Then start a higher-bid camp, and BLACKLIST all the placements that are getting good traffic in the low-bid camp. Run traffic to that camp to identify the placements that are getting good traffic. Then start a higher-bid camp than the other 2, and blacklist all the placements that are getting good traffic in both other camps. You can decide how many of these camps to set up. The idea is for the higher-bid camps to trigger higher-competition placements, which are typically of higher-quality (or else competition wouldn't be so high) so that you're not overbidding for shitty placements or underbidding for great placements. You would then analyze and cut placements for each camp.

    Thanks for the nice discussion Holger! Feel free to continue. Like I said, I'll probably write a detailed post on the strategies outlined above in a separate thread in the near future.

    You're absolutely correct - it's a chicken+egg or catch 22 situation.

    Some options to combat this issue:

    1)Start cutting placement even while you're testing. Then when you're done testing, retest some/all of the placements you've cut before, knowing/hoping some of those will be profitable then.

    This is essential if you're getting a lot of traffic from a lot of placements, where if you don't start cutting right away, you can stand to lose a lot of money unnecessarily.

    2)Avoid cutting placements while you're testing, and only cut the really big losers. Only start cutting more after testing is done.

    This approach is recommended if you don't have a lot of traffic to start with - in which case the more you cut, the slower your testing will go - so you'd want to avoid cutting unless absolutely necessary - like when a placement is an obvious big money-suck.

    As for using best creatives from TJ on the new networks - yes that would be a good start. But keep in mind that each network has different sites, so that some angles will work well on some sites and not others, and therefore work well on one network but not another (depending on what kinds of sites the network chiefly consists of).

    For the really big sites/placements, it would be worth your time to customize your banner designs for each. The look-and-feel for example - you can use the same color scheme at the target site to make your banners blend in better, to improve CTR (please check that this is not against the networks' TOS though - they don't all allow this tactic). And of course, targeting different adult categories (e.g. BBW, Ebony...) using corresponding niche angles is always a good technique that will help you avoid the worst of the competition.

    Totally agree with Amy, on cutting placements while testing.

    Receiving a significant volume of impressions from a specific placement that is not generating a single click will lower down the chances to get to better ones while wasting good amount of the testing budget. Keep a close look on the number of impressions and banner clicks. When testing a new GEO on RON traffic I usually let a placement generate like 2k-3k impressions and kill it if there were no banner clicks despite of the spent. This way I could get to better performing placements faster, even though it might look a bit aggressive.

    LP CTR is quite important as well, since there are quite a lot of poor quality or bot placements that will just click the banner and never click on your landing page.

    Since you're running banner you can easily run the bot test caurmen described in this post when using a landing page.

    As you progressively optimize your camp you will find the best performing banner/lander/offer combination per traffic source as performance may vary from one source to another.
